### search-collision-reducer (pick order 7)
- mission: M2 (research support) + M3 (quality filter)
- status change: queued → done
- what was built: `tools/search-collision-reducer/` — search result prefilter with 3-component scoring
  - **relevance scoring**: exact phrase match in body/links (0.95/0.90), token ratio tiers, 6 relevance levels
  - **collision detection**: tokenizes query + username, flags results where match comes only from username overlap. Hard collision (all tokens username-only, score ≥ 0.75) → auto-discard. Soft collision (any overlap + low body relevance) → downrank.
  - **novelty scoring**: fresh authors = 1.0, seen authors with new signal content = 0.4, stale seen authors = 0.1. Anti-circular: for seen authors, merely containing query tokens in body does NOT count as new content — only signal markers (API/CLOB/repo refs) or link targets qualify.
  - **ranking**: combined = relevance×0.50 - collision×0.30 + novelty×0.20, sorted descending
  - **discard logic**: keep=false for hard collisions, soft collisions with low relevance, stale seen authors, or combined score < 0.30
  - **externalized config**: all weights, thresholds, stop words, signal patterns in rules.json
