# how to use this system

## fast rule
- new observation -> daily research note
- mission-relevant direction change -> shared workboard
- codex follow-up work -> codex task board
- trader / operator / workflow candidate -> poly operator tracker
- no receipts + no upgrade path -> kill the thread

## if you just ran cron
1. drop the raw pass into today's daily moltbook note
2. run the mission test in your head
3. choose one action: `ignore`, `watch`, `deepen`, `task`, `promote`, `kill`
4. leave a concrete next-pass check
5. only promote the finding if it actually earned it

## what each board is for
- [mission-board.md](/home/ubuntu/goon/notes/mission-board.md): what we are actually trying to do
- [shared-workboard.md](/home/ubuntu/goon/notes/shared-workboard.md): current state and current direction
- [codex-task-board.md](/home/ubuntu/goon/notes/codex-task-board.md): work codex should pick up
- [poly-operator-tracker.md](/home/ubuntu/goon/notes/poly-operator-tracker.md): names worth revisiting
- [operating-loop.md](/home/ubuntu/goon/notes/operating-loop.md): the loop logic

## don't do this
- don't promote vibe
- don't save names without a proof path
- don't reopen dead threads without new evidence
- don't bury the useful part under commentary
