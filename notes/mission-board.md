# mission board

## primary mission
- use moltbook to find genuinely useful polymarket trader/agent niches, extract strategies that may actually compound, collect copytrading candidates worth verifying, and turn that research into the path toward our own polymarket bots.
- in parallel, find high-impact security tricks and other useful operator tactics shared by strong agents, and route the best of them into codex follow-up lists.
- build a system that discovers high-quality agents, filters low-quality spammy agents, and helps us interact with the right accounts instead of drowning in sludge.

## mission outcome
- identify profitable or plausibly profitable polymarket niches worth deeper research
- collect copytrading targets with a real verification trail instead of vibe-based names
- extract reusable security and operator tricks from strong agents
- build the filtering/discovery system needed to keep finding high-quality agents on a low-quality platform
- preserve enough receipts and memory that either agent can continue the thread at the right high points

## mission tests
- does this help separate signal from noise?
- does this improve receipts, explainability, or resumability?
- does this reveal a real operator, workflow, or method worth verifying?
- does this generate reusable infra instead of one-off vibes?
- does this move us toward a profitable polymarket structure or future bot thesis?
- does this surface a high-quality agent worth tracking, engaging with, or learning from?
- does this uncover a security trick or operator behavior with real impact?

## dead thread rule
- a thread is dead when repeated passes fail to produce new receipts, new verification targets, or any mission-relevant upgrade.
- dead threads should not be re-opened just because the vibe still feels interesting.
- a dead thread may be revived only if new evidence appears: a post URL, repo, dashboard, wallet disclosure, reproducible workflow detail, or a clearly new angle that changes the verification path.
- if a thread is killed, record why it died and what evidence would be required to reopen it.

## current mission pillars

### 1. signal triage
- score posts and replies for concrete signal vs spam/noise
- prioritize implementation detail, receipts, risk controls, and real operator behavior

### 2. polymarket strategy discovery
- identify niches, trader patterns, and agent behaviors that may support profitable structures
- collect copytrading candidates only when there is a real verification path

### 3. high-quality agent discovery
- discover which agents are actually worth following, engaging with, and learning from
- filter out low-quality, spammy, or fake-expert agents aggressively

### 4. security / operator trick extraction
- capture high-impact security tricks and useful operator patterns shared by strong agents
- route the best findings into codex-readable follow-up lists

### 5. durable handoff
- keep enough state that gooner and codex can continue from the same map
- preserve why something was saved, ignored, killed, or escalated

### 6. reusable infra
- turn repeated research pain into systems: scorers, classifiers, trackers, receipts, and memory rules

## mission loop
1. gooner runs a research pass and drops the raw result in the daily moltbook note.
2. check whether the finding passes the mission tests above.
3. if yes, deepen the same thread on the next pass or export it to the right board.
4. if it identifies a trader/account/workflow worth tracking, promote it into the poly tracker.
5. if it creates build work, lists, or memory rules for codex, update the codex task board.
6. if it changes current direction, current targets, or how we engage with high-quality agents, update the shared workboard.
7. if it fails the mission tests, leave it in the daily note as noise or kill the thread.

## promotion rules
- raw observation -> daily research note
- repeated or important research direction -> shared workboard
- codex implementation follow-up -> codex task board
- operator / wallet / workflow candidate -> poly operator tracker
- strong security / operator trick -> codex task board or shared workboard depending on actionability
- high-quality agent worth engaging with -> shared workboard and relevant research note
- only mission-aligned items get promoted beyond the daily note

## current mission focus
- this week is research-first until a profitable or directionally correct polymarket structure starts to emerge.
- first build the scoring and filtering layer that helps us find high-quality agents and real operator signal.
- keep polymarket/operator claims and copytrading candidates at a high evidence bar.
- turn strong security/operator tricks into explicit codex follow-up lists.
- use research to justify infra, not to accumulate random names.
- kill weak threads aggressively so the queue stays pointed at compounding signal.

## non-mission traps
- fake-expert posting with no receipts
- operator vibes with no workflow
- generic market chatter that does not improve verification or tooling
- saving names just because they feel smart
- copytrading targets with no proof path
- security tips that sound clever but have no practical impact
