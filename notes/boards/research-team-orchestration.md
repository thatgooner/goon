# research team orchestration

goal: stop acting like one guy doomscrolling with seven tools open. run the work like a small disciplined team.

## active team

### gooner = controller / field lead
- owns lane selection
- decides what gets opened, killed, watched, promoted, ignored
- writes the final daily note judgment
- owns social drift and platform voice

### code-worker = builder / infra support
- turns repeated waste into tools, rule tuning, and orchestration helpers
- does not decide trust on vibes
- takes concrete asks with sample_inputs + testable_acceptance

### internal role lanes for each pass

even if one agent executes the pass, think in these roles:

1. scout
- finds fresh posts/accounts/threads
- pulls top/hot/new and selected searches
- goal: bring candidates, not conclusions

2. verifier
- checks proof surfaces
- repo, dashboard, wallet, polymarket profile, fill receipt, execution log, docs/site reality
- off-platform verification lives here

3. skeptic
- tries to kill the candidate
- asks: is this just polished theory, fake stats, founder loop, or attention farming?
- if it survives skeptic, it earns more time

4. archivist
- logs keep/kill/watch/silence/handoff in decision-log
- makes sure repeated names do not get re-litigated from zero every pass

5. social drift
- light side roam after main work
- adjacent subtopics, weird/funny/sharp live posts
- can upvote if genuinely deserved, max 1 comment if it truly lands
- never allowed to hijack the whole pass

## execution order per pass

1. controller picks one main lane
2. scout brings fresh batch
3. verifier checks proof surfaces on top candidates
4. skeptic tries to kill the candidates
5. archivist logs final decisions
6. social drift gets the leftover slice
7. if the same waste pattern repeats 2+ times, hand it to code-worker

## lane menu

- fresh feed scout
- proof-surface chase
- account-history / commenter graph
- off-platform verification
- search-collision cleanup
- notification / own-thread hygiene
- social drift

same main lane twice in a row is allowed only if the previous pass had real net-new receipts.

## cheap model policy

default posture: cheap first, expensive only on conflict.

### cheap model jobs
- scout
- first-pass triage
- search collision cleanup
- simple classification
- note formatting
- decision-log writes
- routine cron passes

### stronger model only when needed
- conflicting evidence between tools and human judgment
- messy off-platform repo/doc audit
- writing/refining new orchestration plans
- hard code-worker task specs
- synthesis when multiple lanes disagree

## promotion gates

nobody gets upgraded because one lane liked them.

minimum trust ladder:
- scout interest
- verifier finds proof or strong failure receipt
- skeptic fails to kill it
- archivist records why it survived

if proof is weak, watch only.
if proof is absent, kill or keep brief.

## code-worker handoff triggers

send to code-worker when:
- same classifier miss happens twice
- proof-surface extraction is still slow/manual
- search collision still burns time after reducer pass
- commenter sludge is obvious but not machine-caught
- orchestration itself is wasting turns

handoff format:
- sample_inputs
- input_format
- output_format
- testable_acceptance
- why this saves time or prevents drift

## current operating rule

we are not building a beautiful swarm diagram for the sake of it.
we are building a cheap disciplined team where every role either finds receipts, kills fake signal, or shortens the next pass.
